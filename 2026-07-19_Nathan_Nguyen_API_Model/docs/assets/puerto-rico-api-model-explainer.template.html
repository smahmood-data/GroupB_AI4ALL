Puerto Rico dengue forecast, explained.

This page covers only the new Puerto Rico-wide system trained from public health and weather APIs. It does not use the historical DengAI/Kaggle city dataset or the San Juan/Iquitos notebook results.

Inputs

Two live API sources

The operational target is the entire island. The pipeline uses the same source families for historical training and current scoring.

01

Puerto Rico Department of Health

Daily PCR and IgM dengue counts are added together. Hospitalizations are retained as a supporting signal, and only complete Monday–Sunday weeks are kept.

Official API documentation ↗
02

Open-Meteo

Temperature, rainfall, rainy hours, dew point, shallow soil moisture, humidity, and evapotranspiration are sampled across six island locations. Daily rain also becomes weekly heavy-rain frequency and longest-dry-spell features.

Weather API documentation ↗
03

Time-safe weekly features

Weather uses independent lags and rolling summaries. Heavy-rain frequency and longest dry spell feed the MAE-focused case models only; held-out tests showed they improved case error but weakened the alert classifier. Case and hospitalization features are anchored to an exact prior week or the latest delayed report.

No future validation values are used
Architecture

One pipeline, two objectives

The system selects exact cases, delayed cases, or weather only based on what truly exists at forecast time, then produces a case estimate and a separate outbreak alert.

01

MAE-focused case model

P50 is a median forecast from a gradient-boosted quantile regressor. Exact reports use a learned change; delayed reports keep the old count as P50 when held-out corrections cannot beat that persistence baseline.

Primary score: mean absolute error
02

Recall-focused alert model

A calibrated logistic classifier estimates outbreak probability. Its alert gate is selected on later held-out weeks to target at least 80% recall.

Primary policy: catch outbreaks, accept more alerts
Data availability

The three models, with examples

The router uses the strongest case history that genuinely exists: exact recent cases first, then delayed cases, then weather only.

01

Weather only

Definition: Uses weather patterns and seasonality without any usable case history.

Example: To predict July 13 when no continuous case reports are available, the model relies on temperature, rain, humidity, soil moisture, and related weather lags.

Always available · least accurate
02

Weather + delayed cases

Definition: Uses weather plus the newest case report available, even when that report is several weeks old. The model is told the report's age.

Example: To predict July 13 when the newest report is May 18, it uses the May 18 count, the eight weeks leading to it, hospitalizations, current weather, and an eight-week report age. In the committed example, P50/P80/P90 are 11/33/51 cases.

Realistic during reporting delays
03

Weather + exact recent cases

Definition: Uses weather plus all eight case-report weeks immediately before the prediction week.

Example: To predict July 13, reports must be available for every week through July 6 and back through May 18. Those exact recent trends give the model its strongest case-count signal.

Most accurate · requires current reports

Interpret carefully

What the results do not prove

This is a research forecasting system, not an official Puerto Rico Department of Health outbreak declaration.

Delayed P50 is conservative

At an eight-week reporting delay, learned P50 corrections did not beat carrying the latest report forward. The delayed route therefore uses that report as P50, while weather and history still shape the classifier and upper range.

Upper quantiles need calibration

P80 and P90 do not yet contain 80% and 90% of held-out outcomes. Treat them as scenario estimates, not statistical guarantees.

The label is statistical

An outbreak means cases meet or exceed the historical 75th percentile for that seasonal neighborhood using earlier years only. It is not a clinical definition.
